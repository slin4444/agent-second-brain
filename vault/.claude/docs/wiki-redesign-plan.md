# Wiki Redesign Plan

Переход от Filing Clerk модели к Writer модели (Personal Knowledge Wiki).

## Контекст

**Пользователь:**
- Профессия: разработка документов (ГОСТ, методики испытаний), проект ВСМ-1
- Хобби: изучение ИИ-инструментов для личной эффективности
- Личные проекты: помощь друзьям/родственникам (SEO, продвижение финсоветника, аудит бизнес-процессов)
- Главный use case: разобраться в своих мыслях и идеях

**Проблема с текущей системой:**
- A. Куча мелких файлов-осколков вместо цельных статей
- Б. LLM механически раскладывает по папкам, не думает о связях
- В. Нет памяти между сессиями (каждый раз с нуля)
- Г. Нет способа запросить систему ("что я думаю о X?")
- Д. Нет навигации, только дерево файлов

## Целевая структура vault

```
wiki/
├── ai/
│   ├── tools/       ← Статьи про инструменты (Claude, GPT, Cursor, etc.)
│   └── workflows/   ← ИИ-воркфлоу под конкретные задачи (для всех проектов)
├── projects/
│   ├── seo.md       ← Доменные знания: что работает в SEO, паттерны
│   ├── finance.md   ← Доменные знания: продвижение финсоветника
│   └── bizaudit.md  ← Доменные знания: аудит бизнес-процессов
├── people/          ← [имя].md = профиль человека + контекст его проекта
│   └── [name].md    ← Кто + ЦА + тон оф войс + стиль + история
├── patterns/        ← Паттерны поведения и мышления
├── beliefs/         ← Ценности, принципы
└── learnings/       ← Кросс-доменные выводы
```

**Удаляется:** `thoughts/ideas/`, `thoughts/reflections/`, `thoughts/learnings/`, `thoughts/projects/`

**Остаётся без изменений:** `business/`, `projects/` (операционные), `daily/`, `goals/`

## Шаблон карточки человека (wiki/people/)

```markdown
# [Имя]

## Кто
[кем приходится, чем занимается]

## Проект: [название]
**Цель:** ...
**ЦА:** [кто покупатель/читатель, их боли и мотивация]
**Тон оф войс:** [формальный/дружеский/экспертный...]
**Стиль:** [ключевые слова, чего избегать]
**Ключевые месседжи:** ...

## История взаимодействия
[что делали, что сработало, что нет]

## Ссылки
→ [[projects/clients/name]] ← операционные задачи
```

## Принципы wiki-статей

- Писать как Wikipedia, не как дневник
- Статья = накопленное понимание темы, не хроника событий
- Максимум 2 прямые цитаты из записей
- Директория определяется темой: AI инструмент → `wiki/ai/tools/`, SEO паттерн → `wiki/projects/seo.md`
- Статьи живут долго и пополняются (не перезаписываются)

## Разница: Filing Clerk vs Writer

| | Сейчас (Filing Clerk) | Цель (Writer) |
|---|---|---|
| Мысль об SEO | → `thoughts/ideas/seo-tip-2026-04-07.md` | → добавляется в `wiki/projects/seo.md` |
| Мысль о Claude | → `thoughts/learnings/claude-note.md` | → добавляется в `wiki/ai/tools/claude.md` |
| Запрос "что я знаю о SEO?" | Нет ответа | claude-claw читает `wiki/projects/seo.md` |
| Задача для Ивана | Claude не знает его ЦА | Claude читает `wiki/people/ivan.md` |
